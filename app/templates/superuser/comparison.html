SkillMatch Pro Strengths

Deterministic & Reproducible
Same inputs always produce the same team recommendation. Critical for auditing and compliance. LLMs generate different outputs each time.
Mathematical Constraint Enforcement
Hard constraints are enforced with absolute certainty. If max workload is 80%, nobody above 80% is ever recommended. LLMs can "forget" or approximate constraints.
Multi-Role Conflict Prevention
The engine guarantees no person is assigned to two roles. LLMs may suggest the same person for multiple roles if not explicitly told not to.
Academic Rigour (TOPSIS + MCDA)
Uses peer-reviewed mathematical methods. Ideal/worst solution distance calculation with vector normalisation — not heuristic "gut feeling."
Complete Auditability
Every decision is stored with full score breakdowns, timestamps, rejection reasons, and parameter snapshots. Required for enterprise compliance.

LLM Limitations for This Task

Prompt Sensitivity
Small changes in how you phrase the request can produce completely different team recommendations. There's no stable, configurable interface.
No Persistent Database
You must paste all 150 employee profiles into every prompt. LLMs can't query a live database, track changes, or remember past allocations.
No Formal Scoring Method
LLMs approximate scoring based on pattern matching, not formal mathematical methods. They cannot compute TOPSIS closeness coefficients or Monte Carlo simulations.
No Explainability Guarantee
LLMs can explain their reasoning, but it may be post-hoc rationalisation rather than the actual decision process. SkillMatch Pro's explanations are computed from real scores.
Context Window Limits
With 150+ employees and detailed profiles, LLMs may hit token limits, lose track of early data, or produce truncated results.

How to Run the Comparison Test

Step 1: Export the employee dataset from exports/employees_dataset.json (generated during seeding)
Step 2: Create a project in SkillMatch Pro with specific roles and constraints. Run the engine. Save the results.
Step 3: Paste the same dataset + requirements into ChatGPT, Gemini, and Grok. Ask each to form a team.
Step 4: Compare: Did the LLM respect all hard constraints? Did it avoid double-assigning? Can it explain scores? Run it again — is the answer the same?
Expected Result: SkillMatch Pro will produce a deterministic, fully-explained, constraint-compliant team. LLMs will produce varied, partially-explained results that may violate constraints or assign the same person to multiple roles.

Mathematical Foundation

TOPSIS
Technique for Order Preference by Similarity to Ideal Solution (Hwang & Yoon, 1981). Ranks alternatives by computing their geometric distance from the ideal best and ideal worst solutions in a normalised, weighted decision matrix.
Skill Decay Model
Exponential decay function: score = e^(-λt) where λ = ln(2)/24 months. A skill unused for 2 years drops to 50% effectiveness. Based on Ebbinghaus forgetting curve principles adapted for professional skills.
Monte Carlo Simulation
200 iterations with ±15% random weight perturbations. Measures how frequently the top candidate remains #1 across all variations. High consistency = high confidence in the recommendation.
Pareto Optimality
Three weight profiles generate teams optimised for different trade-offs (skill vs cost vs balance). No single team dominates all others on every criterion — the manager chooses which trade-off suits their project needs.
